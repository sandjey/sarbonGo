Техническое задание и флоу
Регистрация и проверка документов водителей (v1.0)

1. Общая цель
Данный документ описывает процесс регистрации водителей и подтверждения документов в мобильном приложении.
Система состоит из двух этапов:
Этап 1 — базовая регистрация
Этап 2 — проверка документов перед созданием предложения (KYC)

2. Этап 1 — Базовая регистрация
2.1 Флоу
PHONE → OTP → NAME (+ оферта) → PERMISSIONS (гео + уведомления) → VEHICLE (можно пропустить) → DONE → Home

2.2 Шаги
1. Ввод телефона (PHONE)
Пользователь вводит номер телефона
Получает код подтверждения в Telegram
2. Ввод кода (OTP)
Ввод 6-значного кода
Проверка на сервере
Получение токенов
3. Имя + оферта (NAME)
Поле: Имя и фамилия
Чекбокс: принятие оферты
Кнопка "Далее" активна только при галочке
4. Разрешения (PERMISSIONS)
Геолокация
Push-уведомления
Пользователь может разрешить или отказаться
5. Тип транспорта (VEHICLE)
Поле 1: Грузовик / Тягач
Поле 2: Прицеп / Полуприцеп (зависит от поля 1)
Кнопки: "Пропустить" / "Далее"
6. Завершение (DONE)
Доступ в приложение
Возможность просматривать грузы

2.3 Правила доступа к "Ближайшим грузам"
Вкладка Nearby доступна только если:
geo_permission = GRANTED
last_lat и last_lng заполнены
truck_type заполнен
Если условия не выполнены — показывается заглушка.

3. Этап 2 — Проверка документов (KYC)
3.1 Назначение
Этап запускается при попытке сделать предложение по грузу.
Цель — подтвердить личность водителя и транспорт.

3.2 Флоу
START OFFER → DRIVER ID → POWER UNIT DOC → TRAILER DOC (если нужен) → KYC DONE → OFFER FORM

5) Основные бизнес-флоу
5.1 Создание груза и модерация
MANAGER создаёт Cargo → CREATED
отправляет на модерацию → ON_MODERATION
модерация:
approve → APPROVED (и/или сразу OFFER_STAGE)
reject → REJECTED (менеджер правит и снова)
5.2 Сценарий A: Carrier → Driver → Dispatcher → IN_EXECUTION
Cargo APPROVED/OFFER_STAGE
CARRIER отправляет offer своему водителю (CARRIER_TO_DRIVER) → Offer SENT
DRIVER принимает → Offer ACCEPTED
создаётся Execution в статусе PENDING_DISPATCHER_CONFIRM
DISPATCHER подтверждает → Execution=IN_EXECUTION, Cargo=IN_EXECUTION
Cargo становится неактивным в админке
Проверки:
водитель должен относиться к перевозчику (если у вас есть такая связь) или быть разрешён как “свой водитель”
диспетчер должен быть в carrier_dispatcher_links этого перевозчика
диспетчер должен иметь driver_dispatcher_link на этого водителя
5.3 Сценарий B: Driver → Dispatcher → Driver → IN_EXECUTION
Cargo APPROVED/OFFER_STAGE
DRIVER отправляет application/offer на cargo (DRIVER_TO_DISPATCHER)
DISPATCHER отвечает оффером водителю (DISPATCHER_TO_DRIVER) → SENT
DRIVER принимает → ACCEPTED
Execution=IN_EXECUTION, Cargo=IN_EXECUTION
Cargo становится неактивным в админке
Проверки:
диспетчер имеет driver_dispatcher_link на этого водителя
диспетчер принадлежит перевозчику груза (carrier_dispatcher_link)

User Flow для фриланс-диспетчера на платформе Sarbon
1. Кто такой фриланс-диспетчер
Фриланс-диспетчер — независимый специалист, который может работать в нескольких компаниях (Shipper, Broker, Fleet) как наёмный сотрудник с ролью Dispatcher или TopDispatcher, либо создать собственную компанию типа Broker и работать самостоятельно, привлекая своих водителей.
2. Регистрация и выбор пути
Регистрация: Диспетчер регистрируется на платформе как физическое лицо (email/телефон, пароль).
После регистрации перед ним открываются два пути:
Работа в чужой компании – ожидание приглашения от существующей компании.
Самостоятельная работа – создание собственной компании типа Broker.
3. Путь 1: Работа в чужой компании
Диспетчер получает приглашение по электронной почте от компании, в которой ему назначают роль Dispatcher или TopDispatcher.
Переходит по ссылке-приглашению, принимает его и становится сотрудником компании.
При входе в систему теперь видит эту компанию в списке доступных.
4. Путь 2: Самостоятельная работа (собственная компания)
Диспетчер создаёт компанию типа Broker (экспедитор), заполняя название, ИНН (опционально) и контактные данные.
Автоматически становится владельцем (Owner) компании.
При необходимости назначает себе роль Carrier (перевозчик), чтобы иметь возможность делать ставки на грузы (владелец и так имеет все права, но для явности можно указать).
Приглашает своих водителей в компанию с ролью Driver (по email). Водители принимают приглашения и становятся сотрудниками компании.
После этого компания готова к работе: есть владелец-диспетчер и база водителей.
5. Вход в систему и выбор компании
Диспетчер входит на платформу (через веб-интерфейс Sarbon.me) с помощью логина и пароля или OTP.
Если у него есть доступ к нескольким компаниям (например, собственная и ещё одна, куда пригласили), система показывает список доступных компаний.
Диспетчер выбирает компанию, в контексте которой будет работать в данный момент. Все последующие действия (просмотр грузов, создание ставок, чаты и т.д.) выполняются от имени этой компании и с учётом его роли в ней.
6. Основные рабочие сценарии
После выбора компании диспетчер попадает на дашборд, откуда может выполнять два основных вида деятельности: выступать в роли заказчика (создавать грузы) или в роли перевозчика (искать грузы и делать ставки). Эти сценарии могут выполняться параллельно.
6.1. Сценарий «Диспетчер как заказчик» (создание груза)
Доступность: Только если в выбранной компании у диспетчера есть роль, позволяющая создавать грузы (Owner, CEO, Top Manager, Manager). Для наёмного диспетчера с ролью Dispatcher эта функция обычно недоступна.
Действия:
Диспетчер создаёт новый груз, заполняя все необходимые параметры (маршрут, вес, даты, тип транспорта, цена и т.д.).
Публикует груз – он получает статус searching (поиск перевозчика).
Ожидает поступления ставок от перевозчиков (водителей, других диспетчеров).
Просматривает список поступивших ставок, видит информацию о перевозчике и предложенной цене.
Принимает решение:
Принять ставку – тогда автоматически создаётся рейс (статус assigned). Груз переходит в статус assigned.
Отклонить ставку – ставка помечается как отклонённая, груз остаётся в поиске.
После принятия ставки диспетчер отслеживает выполнение рейса (видит статусы, может общаться в чате с водителем).
Когда рейс завершён (статус completed), груз автоматически закрывается.
6.2. Сценарий «Диспетчер как перевозчик» (поиск грузов и ставки)
Доступность: Если в компании есть роль Carrier (или владелец компании Broker). Диспетчер может действовать как перевозчик, даже если он одновременно является заказчиком.
Действия:
Диспетчер ищет доступные грузы (с фильтром по статусу searching).
Выбирает подходящий груз и создаёт ставку, указывая свою цену и комментарий.
Ожидает решения грузовладельца (той компании, которая опубликовала груз).
Если ставка принята:
Система создаёт черновик рейса со статусом pending_driver (ожидает подтверждения водителя). Груз пока не считается назначенным окончательно.
Диспетчер выбирает одного из своих водителей (из списка сотрудников компании с ролью Driver) и отправляет ему предложение выполнить этот рейс.
Водитель получает уведомление в мобильном приложении Driver App, видит детали рейса и может:
Подтвердить – тогда рейс переходит в статус assigned, закрепляется за водителем, и он может начинать выполнение.
Отклонить – предложение снимается, диспетчер уведомляется об отказе и может предложить рейс другому водителю.
Если водитель не отвечает в течение установленного времени (тайм-аут), предложение автоматически отменяется, и диспетчер может выбрать другого водителя.
Если ставка отклонена грузовладельцем, диспетчер возвращается к поиску других грузов.
После подтверждения водителем начинается выполнение рейса: водитель меняет статусы (loading → en_route → unloading → completed) через мобильное приложение. Диспетчер отслеживает прогресс.
По завершении рейса груз закрывается, статус становится completed.
7. Переключение между компаниями
В любой момент диспетчер может переключиться на другую компанию, в которой он состоит, через специальный переключатель в интерфейсе.
После переключения все данные (дашборд, грузы, ставки, чаты) обновляются в соответствии с контекстом новой компании, и применяются права, соответствующие роли в этой компании.
8. Завершение работы
Диспетчер может выйти из системы (logout) в любой момент.
Сессия завершается, токены становятся недействительными.
Важные замечания
На всех этапах действуют права доступа (матрица ролей). Например, наёмный диспетчер без роли Manager не может создавать грузы.
Все ключевые действия логируются в аудите.
Для работы с водителями необходимо, чтобы они были предварительно приглашены в компанию и имели роль Driver.
Процесс подтверждения водителем обязателен — диспетчер не может принудительно назначить водителя без его согласия.




 User Flow для водителя на платформе Sarbon

Общая концепция
Платформа Sarbon предлагает два режима работы для водителя:
Самостоятельный водитель (OwnerOperator) – после регистрации водитель автоматически получает собственную компанию, может искать грузы, делать ставки и выполнять рейсы.
Наёмный водитель (Driver) – водитель принимает приглашение от компании (Fleet/Broker) и работает по заданиям диспетчера, но также может предлагать грузы диспетчеру для ставок.
Водитель может совмещать оба режима, переключаясь между компаниями в интерфейсе.
Водитель всегда работает с одной машиной (своим транспортным средством). Если груз требует несколько машин, он рассматривается как несколько одинаковых «подгрузов» на одну машину каждый. Водитель может взять только одну машину из этого пула. После создания рейса количество требуемых машин уменьшается на 1, и груз продолжает отображаться в списке доступных, пока все машины не будут назначены. Количество требуемых машин отображается в карточке груза.

1. Регистрация и первый вход
1.1. Ввод номера телефона
Водитель открывает мобильное приложение Sarbon Driver.
Нажимает «Войти» или «Зарегистрироваться», вводит номер телефона в международном формате.
Нажимает «Получить код».
1.2. Подтверждение по SMS
На указанный номер приходит SMS с OTP-кодом.
Водитель вводит код в приложение.
Если код верен, система создаёт учётную запись водителя.
1.3. Запрос разрешений
Приложение запрашивает доступ к геолокации (для сортировки грузов по расстоянию) и push-уведомлениям (для оповещений о событиях).
Водитель разрешает (или отклоняет, но без геолокации не будет работать сортировка по расстоянию).
1.4. Быстрый выбор типа транспортного средства (опционально)
Водитель видит экран с предложением выбрать тип ТС и прицепа. Данные берутся из справочника (например, «тягач + рефрижератор», «грузовик + тент», «фура + бортовой» и т.д.).
Выбор не обязателен. Водитель может:
Выбрать тип ТС – тогда он попадает на главный экран со всеми тремя вкладками.
Пропустить – нажав «Пропустить», он также попадает на главный экран, но вкладка «Подходящие грузы» будет заблокирована (недоступна) с предложением выбрать тип позже.
В любом случае для водителя автоматически создаётся компания типа OwnerOperator(невидимо для него), и он получает роли Carrier и Driver.

2. Главный экран для самостоятельного режима (три вкладки)
«Ближайшие грузы» – все грузы без фильтрации по типу, отсортированные по расстоянию от текущего местоположения водителя (если геолокация разрешена).
«Подходящие грузы» – грузы, отфильтрованные по типу ТС водителя (только те, которые соответствуют его тягачу и прицепу). Если тип ТС не выбран, вкладка недоступна: вместо списка отображается сообщение «Выберите тип машины, чтобы видеть подходящие грузы» и кнопка «Выбрать тип ТС», которая ведёт к экрану выбора типа ТС (в профиле).
«Все грузы» – все доступные грузы без фильтрации, отсортированные по дате публикации (новые сверху).
Каждая карточка груза содержит: маршрут, вес, предлагаемую цену, дату погрузки, а также (если применимо) «Требуется машин: X» (где X – общее количество машин, необходимых для перевозки). Если груз требует несколько машин, это отображается в карточке.
Внизу экрана – нижняя навигация: «Грузы», «Чаты», «Профиль».

3. Профиль и дополнительные разделы
В профиле (доступен через нижнюю навигацию) находятся следующие пункты:
3.1. Мои рейсы
Список всех активных и завершённых рейсов (независимо от того, в какой компании они получены).
При нажатии на рейс открывается карточка с деталями и кнопками смены статусов (для активного рейса).
3.2. Моё авто
Просмотр и редактирование данных о транспортном средстве.
Поля: тип тягача, тип прицепа, госномер, грузоподъёмность, фото техпаспорта, фото паспорта водителя (опционально).
Если тип ТС не был выбран при регистрации, здесь можно его выбрать/изменить в любой момент. После сохранения типа ТС вкладка «Подходящие грузы» на главном экране станет доступна.
3.3. Приглашения от компаний
Список входящих приглашений от компаний (Fleet/Broker).
Каждое приглашение содержит: название компании, возможно, контактные данные.
Кнопки: «Принять», «Отклонить».
При наличии новых приглашений на иконке профиля или на пункте «Приглашения» отображается бейдж.
3.4. Мои компании
Список компаний, в которых состоит водитель (своя и наёмные).
Для каждой компании можно:
Просмотреть информацию.
Покинуть компанию (если это не своя компания). При нажатии проверяется наличие активных рейсов по этой компании. Если они есть, выход блокируется с сообщением: «Вы не можете покинуть компанию, пока не завершены активные рейсы». Если активных рейсов нет, водитель немедленно удаляется из компании.
3.5. Настройки
Общие настройки приложения (уведомления, язык и т.д.).

4. Работа с грузом и создание ставки (самостоятельный режим)
4.1. Выбор груза
Водитель нажимает на карточку груза в любой из доступных вкладок → открывается подробная карточка.
В карточке указано, требуется ли несколько машин для этого груза (например, «Требуется машин: 3»). Если часть машин уже назначена, отображается прогресс (например, «Назначено 1 из 3 машин»).
4.2. Нажатие «Сделать ставку»
Водитель нажимает кнопку «Сделать ставку».
4.3. Проверка наличия полных данных об автомобиле
Система проверяет, заполнены ли обязательные поля в разделе «Моё авто»: госномер, грузоподъёмность (и, возможно, сканы документов).
Если данные отсутствуют или неполные, появляется экран добавления авто:
Предложение: «Для ставки нужны данные машины. Добавить сейчас?»
Два способа:
Сканировать документы – водитель фотографирует техпаспорт и паспорт, система автоматически распознаёт данные.
Заполнить вручную – водитель вводит данные.
После сохранения водитель сразу переходит к форме ставки.
4.4. Форма ставки
Поля:
Цена (обязательно) – водитель указывает цену, за которую готов выполнить перевозку одной машины (если груз требует несколько машин, цена указывается за одну машину).
Комментарий (опционально).
Кнопка «Отправить ставку».
4.5. Отправка ставки и создание чата
Ставка сохраняется со статусом pending.
Автоматически создаётся чат между водителем и грузовладельцем.
В чат отправляется системное сообщение: «[Имя] сделал ставку на груз [название] на сумму [цена].»

5. Переговоры и контрпредложения (в чате)
Все дальнейшие коммуникации по ставке – в этом чате.
Грузовладелец может отправить контрпредложение – в чате появляется карточка с новой ценой и кнопками «Принять» / «Отклонить».
Водитель может:
Принять – ставка (или контрпредложение) считается принятой.
Отклонить – контрпредложение отклоняется, исходная ставка остаётся активной.
Сделать встречное предложение – отправить новую ставку (через кнопку в чате).
Если исходная ставка принята, в чат приходит сообщение: «Ваша ставка принята! Рейс создан.»

6. Принятие ставки и создание рейса (включая грузы с несколькими машинами)
При принятии ставки (исходной или контрпредложения) система:
Переводит ставку в accepted.
Создаёт рейс со статусом assigned, привязывает к водителю и к данному грузу.
Для груза, требующего несколько машин:
Уменьшает счётчик требуемых машин на 1 (например, было «Требуется 3 машины», становится «Требуется 2 машины»).
Если счётчик становится равным 0, груз переходит в статус assigned (все машины назначены). Если ещё остались машины, груз остаётся в статусе searching и продолжает отображаться в списке доступных грузов для других водителей.
Один и тот же водитель не может сделать повторную ставку на этот же груз (система блокирует, так как он уже участвует в рейсе по этому грузу).
Добавляет рейс в раздел «Мои рейсы» водителя.
Отправляет уведомление в чат.

7. Выполнение рейса
Водитель открывает «Мои рейсы», выбирает активный рейс.
В карточке рейса кнопки смены статусов:
«Начать погрузку» → loading
«Отправиться в путь» → en_route
«Прибыл на разгрузку» → unloading
«Завершить рейс» → completed
Статусы видны грузовладельцу и диспетчеру (если есть). Для груза с несколькими машинами грузовладелец видит прогресс по каждой машине отдельно (в списке рейсов).

8. Приглашения от компаний и переход в наёмный режим
8.1. Получение приглашения
Если компания отправляет приглашение, оно появляется в разделе «Приглашения» профиля.
Водитель получает push-уведомление: «Компания [Название] приглашает вас стать водителем».
8.2. Принятие приглашения
Водитель заходит в раздел «Приглашения», нажимает «Принять».
Система добавляет водителя в компанию с ролью Driver. Теперь у водителя есть как минимум две компании: своя (OwnerOperator) и наёмная.
В интерфейсе появляется переключатель компаний (например, в верхней части экрана).
8.3. Переключение на наёмную компанию
При выборе наёмной компании в переключателе главный экран меняется:
Вместо трёх вкладок появляются две вкладки:
«Предложения» – рейсы, отправленные диспетчером этой компании (статус pending_driver). Водитель может принять или отклонить.
«Грузы» – доступные грузы (те же, что и в самостоятельном режиме). Но вместо кнопки «Сделать ставку» здесь кнопка «Предложить диспетчеру». При нажатии диспетчер компании получает уведомление с этим грузом и может рассмотреть его для ставки.
Кнопка «Сделать ставку» в карточке груза отсутствует.
8.4. Работа в наёмном режиме
Водитель может принимать предложения от диспетчера или предлагать грузы диспетчеру.
Активные и завершённые рейсы доступны в разделе «Мои рейсы» (в профиле).
Чаты – с диспетчером и грузовладельцем.
8.5. Возврат к самостоятельной работе
Водитель может в любой момент переключиться на свою компанию (OwnerOperator) через переключатель.
При переключении главный экран снова становится трёхвкладочным списком грузов, доступны создание ставок и т.д.
Рейсы, полученные в наёмном режиме, остаются в разделе «Мои рейсы» и выполняются независимо.
8.6. Выход из компании
Водитель может покинуть наёмную компанию в любой момент (кроме случаев, когда у него есть активные рейсы по этой компании).
Для этого в разделе «Мои компании» он выбирает нужную компанию и нажимает «Покинуть компанию».
Система проверяет наличие активных рейсов (статусы assigned, loading, en_route, unloading) по этой компании. Если такие рейсы есть, выводится сообщение: «Вы не можете покинуть компанию, пока не завершены активные рейсы». Выход блокируется.
Если активных рейсов нет, водитель немедленно удаляется из компании (запись в user_company_roles удаляется). Компания исчезает из переключателя. Водитель получает уведомление об успешном выходе.
8.7. Отказ от приглашения
Водитель может отклонить приглашение, нажав «Отклонить» в разделе «Приглашения». Приглашение удаляется.

9. Уведомления (push и in-app)
Водитель получает уведомления о:
Новом приглашении от компании.
Новом предложении рейса от диспетчера (для наёмного режима).
Ставка принята (рейс создан).
Контрпредложении.
Изменении статуса рейса.
Новом сообщении в чате.

10. Технические детали (для разработки)
Автоматическое создание компании – при регистрации создаётся компания типа OwnerOperator. Водитель – Owner, Carrier, Driver (записи в user_company_roles).
Приглашения – таблица invitations с полями phone, company_id, role_id (Driver), status (pending/accepted/expired). При принятии создаётся запись в user_company_roles.
Предложения для наёмного водителя – рейсы со статусом pending_driver, у которых company_id наёмной компании, driver_id = null. При принятии driver_id проставляется, статус → assigned.
Кнопка «Предложить диспетчеру» – создаёт уведомление диспетчеру компании (системное или через чат) с ссылкой на груз. Диспетчер может рассмотреть и создать ставку от имени компании.
Чат – создаётся при создании ставки. Участники: водитель и грузовладелец. Для наёмного режима – добавляется диспетчер.
Контрпредложения – через offers.parent_offer_id.
Грузы с несколькими машинами:
В таблицу cargo добавить поле required_vehicles (integer). По умолчанию 1. При создании груза указывается необходимое количество машин.
В карточке груза и в списке отображается текущее значение required_vehicles (сколько ещё машин нужно). Можно также хранить и показывать assigned_vehicles для прогресса.
Ставка всегда создаётся на одну машину.
При принятии ставки:
Создаётся рейс.
required_vehicles уменьшается на 1.
Если required_vehicles становится 0, груз переводится в статус assigned. Иначе остаётся в searching.
Водитель не может сделать повторную ставку на этот же груз (проверка по driver_id в существующих ставках/рейсах).
Выход из компании:
Проверка активных рейсов по driver_id и company_id с соответствующими статусами. Если есть – блокировка.
При успешном выходе запись в user_company_roles удаляется.
